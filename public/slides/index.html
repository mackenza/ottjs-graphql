What is GraphQL?

  • Declarative
  • Product-centric
  • Strongly-typed & Introspective

History

Part of the Facebook tooling (really a loosely-coupled framework) to solve some of the business/scale issues Facebook sees deploying their Web and Mobile applications.

What Problems Does GraphQL Address?

  • Less data transferred - Only the requested data elements are returned from the server, therefore minimizing the amount of data sent.
  • Alleviates "endpoint hell" - there is only one endpoint per GraphQL schema. Changes are done on the schema, not endpoints.
  • Abstraction from actual data structure. Client is aware of what data it needs, but needn't worry about the physical representation of the data store(s).
